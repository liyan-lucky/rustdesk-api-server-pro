# Code of Conduct

## Our Pledge

We want this project to be a respectful and practical space for collaboration.

Participants should treat each other with respect, especially when discussing security issues, compatibility behavior, licensing, deployment failures, or legal risk.

## Expected Behavior

- Be respectful and constructive.
- Keep discussions focused on reproducible technical facts.
- Do not post private data, credentials, tokens, production configs, databases, recordings, or exploit details publicly.
- Respect third-party trademarks, licenses, and attribution requirements.
- Avoid language that suggests this project is officially endorsed by RustDesk, Huawei, HarmonyOS, OpenHarmony, or any other third-party vendor.

## Unacceptable Behavior

- Harassment, threats, abusive language, or discriminatory conduct.
- Publishing secrets, private logs, user databases, recordings, or personal data.
- Submitting copyrighted or proprietary material without permission.
- Using the project issue tracker to distribute malware, credential dumps, exploit kits, or illegal content.

## Enforcement

Maintainers may edit, hide, delete, or lock discussions and may block users when necessary to protect the project and its users.

Security-sensitive reports should follow `SECURITY.md`.
