# i18n V3 Quality Workflow

This project now includes a lightweight i18n quality guard to avoid placeholder-corrupted translations (for example `???`).

## Commands

- Generate report:
  - `pnpm i18n:report`
- Print summary from json:
  - `pnpm i18n:summary`
- Generate fallback todo list (prioritized untranslated keys):
  - `pnpm i18n:todo`
- Compare current summary with baseline:
  - `pnpm i18n:compare`
- Quality gate (fails on suspect/missing/extra keys or low translated ratio):
  - `pnpm i18n:check`
- Strict quality gate (for CI):
  - `pnpm i18n:check:strict`
- JSON output:
  - `pnpm i18n:check:json`
- Single-locale check example:
  - `pnpm i18n:check:ru`
- Sanitize placeholder-corrupted values in target locales:
  - `pnpm i18n:sanitize`
- Dry run sanitize:
  - `pnpm i18n:sanitize:dry`
- Dry run sanitize in json:
  - `pnpm i18n:sanitize:json`
- Guard mode (fails if suspicious text is detected):
  - `pnpm i18n:sanitize:guard`
- Single-locale sanitize example:
  - `pnpm i18n:sanitize:ru`
- CI-friendly bundle:
  - `pnpm ci:quality`
  - `pnpm ci:frontend`

## Threshold Config

File: `scripts/i18n-quality.config.json`

- `maxSuspectPerLocale`: maximum allowed suspect keys per locale
- `maxMissingPerLocale`: maximum allowed missing keys per locale
- `maxExtraPerLocale`: maximum allowed extra keys per locale
- `minTranslatedRatio`: minimum translated ratio by locale
- `ignoreKeyPrefixes`: ignore key prefixes
- `ignoreExactKeys`: ignore exact keys
- `strictOverrides`: stricter limits used in `--strict` mode

Adjust thresholds gradually as locale quality improves.

Baseline file: `scripts/i18n-quality-baseline.json`

- Used by `i18n:compare` for anti-regression checks.

## Output Files

- Coverage report: `docs/i18n-coverage-report.md`
- Quality summary json: `docs/i18n-quality-summary.json`
- Sanitize report json: `docs/i18n-sanitize-report.json`
- Fallback todo (markdown/json): `docs/i18n-fallback-todo.md`, `docs/i18n-fallback-todo.json`

`i18n:check` always refreshes the summary json with pass/fail details by locale.
`i18n:sanitize` always refreshes the sanitize report json with replacement details.

## Home Card Cache TTL

Two env vars are supported for server config card cache behavior:

- `VITE_SERVER_CONFIG_CACHE_TTL_MS` (default `30000`)
- `VITE_SERVER_CONNECTIVITY_CACHE_TTL_MS` (default `10000`)

Set in `.env` / `.env.prod` as needed.

## CI

Workflow: `.github/workflows/frontend-quality.yml`

- Trigger: `push` / `pull_request` (`master` and `main`) + weekly schedule
- Runs strict i18n check, typecheck, build
- Uploads report artifacts
