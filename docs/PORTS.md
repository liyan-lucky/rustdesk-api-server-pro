# 端口与访问路径说明

本文档说明当前项目的端口用途、默认值、访问路径，以及和配置文件的对应关系。

## 1. 总体结构（重要）

当前项目默认是“单 HTTP 端口”架构：

- RustDesk 客户端 API（`/api/...`）
- 管理后台登录与管理接口（`/admin/...`）
- 管理后台前端静态页面（`/`）

都由同一个 Go 服务、同一个 HTTP 监听端口提供。

## 2. HTTP 主端口（核心端口）

配置项位置：`backend/server.yaml`

```yaml
httpConfig:
  port: ":12345"
```

代码默认值（当 `server.yaml` 不存在或读取失败时）：

- `:8080`

说明：

- 示例 `server.yaml` 常见写法是 `:12345`
- 真实生效值以运行目录下的 `backend/server.yaml` 为准

## 3. 路径划分（同端口）

假设 `httpConfig.port = :12345`，服务地址为 `http://127.0.0.1:12345`

### 3.1 管理后台前端页面（静态文件）

- 入口：`/`
- 示例：`http://127.0.0.1:12345/`

说明：

- 由后端通过 `HandleDir("/", staticdir)` 提供
- `staticdir` 路径不正确时，浏览器会打不开后台页面或出现 404

### 3.2 客户端 API（RustDesk 客户端使用）

- 前缀：`/api`
- 示例：
  - `POST /api/login`
  - `GET /api/users`
  - `GET /api/peers`
  - `POST /api/heartbeat`

### 3.3 管理后台接口（管理员使用）

- 前缀：`/admin`
- 示例：
  - `POST /admin/login`
  - `GET /admin/users`
  - `GET /admin/devices`

### 3.4 插件签名兼容接口（单独前缀）

- 前缀：`/lic/web/api`
- 当前兼容端点：
  - `POST /lic/web/api/plugin-sign`

说明：

- 这是为了兼容新版官方客户端/插件流程的请求路径
- 当前为兼容占位实现，不是官方真实签名服务

## 4. SMTP 端口（邮件功能）

配置项位置：`backend/server.yaml`

```yaml
smtpConfig:
  host: "127.0.0.1"
  port: 1025
```

用途：

- 用于后台或系统发送邮件（如某些通知/模板邮件功能）

注意：

- 这是“连接外部 SMTP 服务器”的目标端口，不是本程序额外监听的本地端口
- 当前项目主要业务（客户端 API / 后台页面）不依赖 SMTP 才能启动

## 5. 静态目录与端口的关系（常见误区）

配置项：

```yaml
httpConfig:
  staticdir: "/app/dist"
```

说明：

- `staticdir` 决定“同一个 HTTP 端口下 `/` 页面展示什么”
- 它不是端口配置，但配置错误会让你误以为“端口没启动”
- 常见症状：
  - 端口能访问，但首页空白/404
  - `/api/*` 正常，`/` 不正常

排查顺序：

1. 先确认端口监听正常
2. 再确认 `staticdir` 路径存在并且是前端构建产物目录

## 6. Docker 场景下的端口说明

Docker/Compose 常见配置是“容器内端口 + 主机映射端口”：

- 容器内：由 `server.yaml` 的 `httpConfig.port` 决定（如 `:12345`）
- 宿主机访问端口：由 `docker-compose.yaml` 端口映射决定（如 `8080:12345`）

如果你在宿主机访问失败，请同时检查：

- 容器是否启动成功
- 容器端口映射是否正确
- `server.yaml` 是否在容器内生效
- 防火墙/安全组是否放行

## 7. 建议的端口规划（生产）

最小化部署建议：

- API + 管理后台：一个 HTTP 端口（如 `12345`）
- SMTP：使用现有企业邮件服务（无需本机开放 SMTP 监听）

如果使用反向代理（Nginx/Caddy）：

- 外部统一 `80/443`
- 反代到本程序内网端口（如 `127.0.0.1:12345`）

这样可以简化证书、域名和访问控制配置。
